# Slide Deck Structure (PPTX)

This reference defines the 12-15 slide structure for SMO offer pitch decks. Based on the Engine and Expansion decks.

Before creating the deck, read the **pptx skill** SKILL.md and use `pptxgenjs` for generation. Apply the SMO color palette from `design-system.md`.

## Slide-by-Slide Template

### Slide 1: Title Slide (DARK BG)
- **Background:** `#000000`
- **Content:**
  - Offer name (H1, 44pt, white, bold)
  - Tagline/subtitle (18pt, `#A3A3A3`)
  - 3 key stat callouts in a row (e.g., "3-5x Response Rate | 20% Meeting Rate | 90 Days to ROI")
  - Stats use accent color for numbers, white for labels
  - Brand name/logo placeholder bottom-left
  - Small "Confidential" or date text bottom-right

### Slide 2: The Problem (DARK or WHITE BG)
- **Title:** "The [Category] Problem" or the specific pain
- **Content:** 3-4 problem cards in a grid
  - Each card: bold title + 1-2 line description
  - Red accent indicator (red dot, red left border, or red number)
  - Keep it visceral and specific — not generic "companies struggle with..."
- **Layout:** 2x2 grid or single column with icons

### Slide 3: The Shift / Old vs New (WHITE BG)
- **Title:** "The Shift" or "Old Way vs [Offer] Way"
- **Content:** Two-column comparison table
  - Left column: "Traditional Approach" with pain points (red X markers)
  - Right column: "[Offer Name]" with benefits (green checkmarks)
  - 4-6 comparison rows
  - Use subtle background tints: light red on left, light green on right
- **Alternative:** If not a comparison, show the thesis/framework diagram

### Slide 4: How It Works (DARK BG)
- **Title:** "How [Offer] Works"
- **Content:** 4-stage horizontal process
  - Each stage: number (accent color, large), title, 1-line description
  - Connected by arrows or lines
  - Icons optional but helpful
- **Layout:** 4-column or horizontal timeline

### Slide 5: Detail Slide 1 — Signals/Features (WHITE BG)
- **Title:** Context-specific (e.g., "Signals We Detect", "What's Inside Phase 1")
- **Content:** 6 items in a 2x3 or 3x2 grid
  - Each: icon in colored circle + title + short description
  - Mix of signal types, features, or deliverables depending on offer

### Slide 6: Detail Slide 2 — Execution/Channels (DARK BG)
- **Title:** Context-specific (e.g., "Multi-Channel Execution", "Phase 2: Build")
- **Content:** 3 channel/method cards
  - Each with a colored top border or icon
  - Title + 3-4 bullet features
- **Layout:** 3-column grid

### Slide 7: What's Included (WHITE BG)
- **Title:** "What's Included" or "Full Deliverables"
- **Content:** Comprehensive feature list
  - Organized in 2-3 categories
  - Checkmarks (accent color) for each item
  - Highlight the "surprise and delight" items that competitors don't offer
- **Layout:** 2-column with category headers, or icon grid

### Slide 8: Results / Proof (DARK BG)
- **Title:** "Results That Speak" or "The Proof"
- **Content:**
  - 4 large metric cards: big number (60-72pt, accent) + label
  - Below metrics: 1 case study card with client industry, challenge, result
- **Layout:** 4-column metrics row + full-width case study below

### Slide 9: The Math / ROI (WHITE BG)
- **Title:** "The Math Works" or "ROI Breakdown"
- **Content:** Simple ROI table or calculation
  - Investment row, expected output row, ROI multiplier
  - Use real numbers from typical client scenarios
  - Highlight the key takeaway number (e.g., "8.5x ROI")
- **Layout:** Centered table with accent header row

### Slide 10: Competitive Comparison (DARK BG)
- **Title:** "Why [Offer] Wins" or "The Comparison"
- **Content:** Feature comparison table
  - Rows: 5-7 key capabilities
  - Columns: "[Offer]" vs "In-House SDR" vs "Traditional Agency"
  - Use checkmarks (green) and X marks (red)
  - [Offer] column highlighted with accent background
- **Layout:** Full-width table

### Slide 11: Objections / FAQ (WHITE BG)
- **Title:** "Common Questions" or "What You're Wondering"
- **Content:** 4-6 Q&A pairs
  - Question in bold, answer in regular weight below
  - Address real objections: price, timeline, MENA specifics, integration
- **Layout:** 2-column Q&A grid or single column list

### Slide 12: Vision / System Diagram (DARK BG) — Optional
- **Title:** "The Vision" or "Where This Goes"
- **Content:** System architecture or roadmap diagram
  - Show how the offer fits into a bigger picture
  - Month 1-3 → Month 4-6 → Month 7-12 progression
  - Or: hub-and-spoke diagram showing the integrated system

### Slide 13: Pricing / Investment (DARK BG)
- **Title:** "Investment"
- **Content:** Pricing structure
  - If tiered: 2-3 pricing cards side by side (recommended tier highlighted)
  - If single: one centered card with price breakdown
  - Each card: tier name, price, what's included (bulleted), CTA
  - Setup fees separated from recurring if applicable
  - Performance bonuses or variable pricing noted separately

### Slide 14: Qualifier — Is This For You? (WHITE BG)
- **Title:** "Is [Offer] Right For You?"
- **Content:** Two columns
  - "Perfect fit if..." (green checkmarks) — 4-5 items
  - "Not the right fit if..." (red X) — 3-4 items
  - Be specific and honest — this builds trust

### Slide 15: CTA Slide (DARK BG)
- **Title:** Action-oriented (e.g., "Ready to Build Your Revenue Engine?")
- **Content:**
  - CTA text (e.g., "Book Your Signal Audit")
  - Contact: email, phone, calendar link
  - Small reassurance: "No commitment. 30-minute call."
  - Brand name/logo

## Slide Design Principles

### Color Pattern
Alternate dark and light backgrounds to create visual rhythm:
```
Dark → Dark/White → White → Dark → White → Dark → White → Dark → White → Dark → White → Dark → Dark → Dark
```
Title and CTA slides are always dark. Content-heavy slides alternate.

### Typography in Slides
- Titles: 36-44pt, Bold, max 1 line (2 max)
- Body: 14-16pt, Regular
- Stats: 60-72pt, Bold, accent color
- Captions: 10-12pt, muted color

### Layout Rules
- Minimum 0.5" margins from all edges
- Cards/boxes: 0.3-0.5" internal padding
- Grid gaps: 0.3-0.5" between elements
- Left-align body text (center only titles and stats)
- One visual per slide minimum (no text-only slides)

### What to Avoid
- No accent lines under titles
- No generic blue color schemes
- No walls of bullet text — use cards, grids, tables instead
- No clipart or stock photo placeholders
- No more than 6 bullet points per card/section
- Don't repeat the same layout on consecutive slides

## Speaker Notes
Every slide should have speaker notes with:
- The key talking point (1-2 sentences)
- The transition to the next slide
- Any objection this slide might trigger and how to handle it
