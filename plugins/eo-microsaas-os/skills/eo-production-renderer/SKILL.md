## HTML SLOT INJECTION ALGORITHM

### Step 1: Parse the Template
```javascript
const template = fs.readFileSync(templatePath, 'utf8');
```

### Step 2: Find All Slot Markers
Slot markers follow the pattern: `<!-- {{SLOT:SLOT_NAME}} -->`

```javascript
const slotRegex = /<!-- \{\{SLOT:(\w+)\}\} -->/g;
const slots = [...template.matchAll(slotRegex)];
```

### Step 3: Inject Content
For each slot found, replace the content block that follows the comment with the corresponding content from the design brief.

**Injection rules:**
- The slot comment marks the START of a replaceable block
- The replaceable block is the NEXT HTML element after the comment
- Preserve the element's tag, classes, and attributes
- Only replace the innerHTML (text content)
- If the slot value is an object/array, render it as HTML matching the template's existing structure

### Step 4: Validate
```javascript
// Check no unreplaced slots remain
const unreplaced = output.match(/<!-- \{\{SLOT:\w+\}\} -->/g);
if (unreplaced) {
  console.warn('Unreplaced slots:', unreplaced);
}
```

### Implementation: render-html.js

```javascript
const fs = require('fs');
const path = require('path');

function renderHTML(templatePath, slots) {
  let html = fs.readFileSync(templatePath, 'utf8');

  for (const [slotName, content] of Object.entries(slots)) {
    const marker = `<!-- {{SLOT:${slotName}}} -->`;
    const markerIdx = html.indexOf(marker);

    if (markerIdx === -1) {
      console.warn(`Slot ${slotName} not found in template`);
      continue;
    }

    // Find the next HTML element after the marker
    const afterMarker = html.substring(markerIdx + marker.length);
    const elementMatch = afterMarker.match(/^\s*(<(\w+)[^>]*>)([\s\S]*?)(<\/\2>)/);

    if (elementMatch) {
      const [fullMatch, openTag, tagName, oldContent, closeTag] = elementMatch;
      const startPos = markerIdx + marker.length + afterMarker.indexOf(fullMatch);
      const endPos = startPos + fullMatch.length;

      // Replace only the innerHTML
      const newElement = `${openTag}${content}${closeTag}`;
      html = html.substring(0, startPos) + newElement + html.substring(endPos);
    }
  }

  return html;
}

module.exports = { renderHTML };
```

## PPTX SLOT INJECTION ALGORITHM

The webinar deck cannot be injected via text replacement because PPTX is a binary format. Instead:

### Approach: Script Cloning
1. Read the webinar deck generation script (`generate-webinar-deck.js`)
2. Read the design brief's `webinar_deck` section
3. Replace all hardcoded content variables in the script with brief values
4. Execute the modified script to regenerate the PPTX

### Replaceable Variables in Webinar Deck

| Variable | Design Brief Key | Default Value |
|----------|-----------------|---------------|
| `{{EVENT_HOST}}` | `webinar_deck.EVENT_HOST` | "{{EVENT_HOST}}" |
| `{{OFFER_NAME}}` | `webinar_deck.OFFER_NAME` | "{{OFFER_NAME}}" |
| `{{CASE_1_COMPANY}}` | `webinar_deck.CASE_1_COMPANY` | "{{CASE_1_COMPANY}}" |
| `{{CASE_2_COMPANY}}` | `webinar_deck.CASE_2_COMPANY` | "{{CASE_2_COMPANY}}" |
| `{{CASE_3_COMPANY}}` | `webinar_deck.CASE_3_COMPANY` | "{{CASE_3_COMPANY}}" |
| `{{COMBINED_REVENUE}}` | `webinar_deck.COMBINED_REVENUE` | "{{COMBINED_REVENUE}}" |

### Speaker Notes Override
The design brief can include `speaker_notes_overrides` keyed by slide number:
```json
{
  "speaker_notes_overrides": {
    "S1": "Custom speaker notes for slide 1...",
    "S6": "Custom origin story notes..."
  }
}
```

The renderer replaces `{{SLOT:SN_SPEAKER_NOTES}}` markers with the override content, or removes the marker if no override exists.

## BATCH RENDERING

For multiple students, the renderer accepts a directory of briefs:

```bash
# Single student
node render.js --brief path/to/design-brief.json --templates path/to/reference-templates/ --output path/to/output/

# Batch mode
node render.js --batch path/to/briefs-directory/ --templates path/to/reference-templates/ --output path/to/output/
```

Batch mode iterates through all `*.json` files in the briefs directory and renders assets for each.

## QUALITY CHECKS (POST-RENDER)

After rendering, the skill runs these automated checks:

1. **Slot completion:** Grep all output files for `{{SLOT:` - must return 0 matches
2. **Placeholder detection:** Grep for `{{FIRST_NAME}}`, `{{PRODUCT_NAME}}`, `{{VERTICAL}}` - these are personalization variables for the outreach tools (Instantly, HeyReach), NOT renderer slots. They should remain.
3. **HTML validity:** Basic tag balance check (every `<div>` has `</div>`)
4. **File size sanity:** Each HTML file should be > 1KB and < 500KB
5. **PPTX existence:** webinar-deck.pptx should exist and be > 50KB

## PERSONALIZATION VARIABLES vs SLOT MARKERS

**CRITICAL DISTINCTION:**

| Type | Pattern | Replaced By | Example |
|------|---------|------------|---------|
| Slot Marker | `<!-- {{SLOT:NAME}} -->` | This renderer (from design brief) | `<!-- {{SLOT:HERO_HEADLINE}} -->` |
| Personalization Variable | `{{FIRST_NAME}}` | Outreach tool (Instantly, HeyReach, GHL) | `{{FIRST_NAME}}, saw your post about {{TOPIC}}` |

The renderer MUST NOT replace personalization variables. These are intended for the outreach tools to fill at send time.

Personalization variables to preserve:
- `{{FIRST_NAME}}`
- `{{PRODUCT_NAME}}`
- `{{THEIR_COMPANY_OR_ROLE}}`
- `{{TOPIC}}`
- `{{VERTICAL}}`
- `{{EVENT_OR_CONTEXT}}`

## ERROR HANDLING

| Error | Response |
|-------|----------|
| design-brief.json not found | Exit with error. Tell user to run eo-gtm-asset-factory first. |
| Template file missing | Skip that asset. Log warning in render-log.json. |
| Slot in brief but not in template | Log warning. Do not fail. |
| Slot in template but not in brief | Log warning. Leave original content. |
| PPTX generation fails | Log error. Continue with HTML assets. |
| Output directory not writable | Exit with error. |
